Android面试知识点集锦

Veröffentlicht am 2016-05-01

四大组件

Activity

  1. Activity用于前台交互，通常包含一个窗口
  2. Activity之间通过Intent进行通信
  3. android应用中每一个Activity都必须要在AndroidManifest.xml配置文件中声明，否则系统将不识别也不执行该Activity

Service

  1. Service用于在执行后台任务（仍然是运行在主线程上的）
  2. 开发人员需要在应用程序配置文件中声明全部的service，使用<service></service>标签。
  3. Service通常位于后台运行，它一般不需要与用户交互，因此Service组件没有图形用户界面。Service组件需要继承Service基类。Service组件通常用于为其他组件提供后台服务或监控其他组件的运行状态。
  4. Service用start和bind两种模式

BroadcastReceiver

  1. 用于接受来自外界的广播
  2. 广播接收者的注册有两种方法，分别是程序动态注册和AndroidManifest文件中进行静态注册。
  3. 动态注册广播接收器特点是当用来注册的Activity关掉后，广播也就失效了。静态注册无需担忧广播接收器是否被关闭，只要设备是开启状态，广播接收器也是打开着的。也就是说哪怕app本身未启动，该app订阅的广播在触发时也会对它起作用。

ContentProvider

  1. 内容提供者，使一个应用程序的指定数据集提供给其他应用程序，其他应用可以通过ContentResolver类从该内容提供者中获取或存入数据
  2. 只有需要在多个应用程序间共享数据是才需要内容提供者
  3. ContentProvider实现数据共享
  4. 开发人员不会直接使用ContentProvider类的对象，大多数是通过ContentResolver对象实现对ContentProvider的操作
  5. ContentProvider使用URI来唯一标识其数据集，这里的URI以content://作为前缀，表示该数据由ContentProvider来管理

Handler机制

Handler

通过Handler向消息队列添加消息

Looper

一个线程只能有一个Looper，子线程需要自己创建自己的Looper，Looper启动后会进入一个死循环，不断的从消息队列中取出消息交由Handler处理

MessageQueue

消息队列中包含添加进去点的所有消息，由Looper来调度

Java面试知识点集锦

Veröffentlicht am 2016-04-30

Java面向对象的三个特征

封装

类需要有一个明确的边界，还需要有属性和方法，对于方法的需要有定义和实现，定义是对外公开的，实现是隐藏的，改变方法的实现不会对方法的调用者造成影响

继承

把一些共性放在父类中，子类会继承父类的非私有属性和方法，子类是特殊的父类

多态

多态是运行时的体现，子类的对象可以当做父类的对象来用

多态的原则

  1. 对象类型不变
  2. 只能用引用调用其引用类型中定义的方法
  3. 运行时，根据对象的实际类型去找子类覆盖之后的方法

抽象类和接口的异同

同

  1. 都可以有抽象的方法
  2. 都不能直接实例化，只能依托与实现了其抽象方法的子类来实例化

异

  1. 一个类只能继承一个抽象类，但是可以实现多个接口
  2. 抽象类可以有非抽象的方法和属性，而接口不能有非抽象方法（JDK8可以有默认实现），接口也不能有属性，接口是无状态的，而抽象类可以有状态
  3. 类是用来描叙某一类事物的，而接口是用来描叙具有相同功能的一些列东西

Java中的四中引用

强引用

Object obj=new Object();其中obj就是强引用

强引用只有被gc标记为可回收才会被回收

软引用

SoftReference<Object> obj = new SoftReference<>(new Object()); 其中obj即为软引用

软引用在堆内存不足时，就算引用仍然被持有着也会被释放

弱引用

WeakReference<Object> obj= new WeakReference<>(new Object()); 其中obj即为弱引用

弱引用只要gc在标记时被发现了就会被回收

虚引用

PhantomReference<Object> obj=new PhantomReference<>(new Object(),null); 其中obj即为虚引用

虚引用是最弱的一种引用，你甚至无法获取到对象的引用，虚引用可以在任何时候被gc回收

Java实现线程的几种方式

  1. 继承Thread类，并重写run方法

  2. 实现Runnable接口，并把Runnable的实例作为参数传递给Thread

HashMap原理

  • HashMap内部的存储结构是一个Node数组
  • Node的包含key、value和指向下一个Node的引用
  • put元素时会取key的hashCode并模上当前数组的长度得到下标，存进数组
  • 若是存进去时发生冲突，Java8之前采用链表解决，Java8采用树的方式解决（效率更高）
  • 取元素时也是根据hash值得到下标然后取出，若发生冲突则根据上面的不同方式解决

垃圾回收机制

为什么要有GC

Java不能像C++那样显示回收内存，只能通过GC来回收内存

如何判断对象可被回收

采用可达性分析的方法，若是对象在Java堆中的一些根对象的引用链上则不能被回收，反之则可以被回收

根对象一般包括：方法栈中的对象、本地方法栈中的对象、静态变量等等

回收方法

标记清除法

先根据可达性分析对不可达的对象进行标记，经过几次标记后，对象会被gc回收

回收后会产生大量内存碎片

复制清除法

将内存分为两块，把第一块上存活的对象复制到第二块上，并清空第一块，两块内存交替使用

能够解决回收产生的内存碎片问题，但是不适用与对象存活率较高的情况

一般java堆会分为新生代和老年代，新生代的对象生命周期都比较短，老年代的对象的生命周期比较长，新生代采用复制清除算法，老年代采用标记清除算法，新生代中存活时间长的对象有机会会被转移到老年代

GC种类

  1. Minor GC，回收新生代

  2. FullGC，整个Java堆

二维数组中的查找

题目描述

在一个二维数组中，每一行都按照从左到右递增的顺序排序，每一列都按照从上到下递增的顺序排序。请完成一个函数，输入这样的一个二维数组和一个整数，判断数组中是否含有该整数。

输入描述

array： 待查找的二维数组
target：查找的数字

输出描述

查找到返回true，查找不到返回false

思路

  1. 由于数组是横竖皆有序的，所以右上角的元素的左边都比它小，下面都比它大，定义两个标示dx和dy分别标示当前右上角的x和y坐标
  2. 所以每次都选取数组的右上角的数据和target做比较，如果相等即返回true
  3. 若是target较小，则说明target只会在这列的左边，dx--，重复上一步
  4. 若是target较大，则说明target只可能在下面，dy++，重读第二部
  5. 直到右上角和左上角重合还没找到说明不存在并返回false

求旋转数组的最小数字

题目描叙

把一个数组最开始的若干个元素搬到数组的末尾，我们称之为数组的旋转。输入一个非递减序列的一个旋转，输出旋转数组的最小元素。例如数组{3,4,5,1,2}为{1,2,3,4,5}的一个旋转，该数组的最小值为1。

思路

由于旋转数组是由两个有序的数组组合而来，则可以使用二分查找的方法

  1. low=0，high指向数组最后一位
  2. 取mid=（low+high）/2并对比array[0]和array[mid]
  3. 若是前者大或是相等则说明mid处在第一个数组上，则取low=mid
  4. 若是后者大则是说明mid处在第二个数组上，去high=mid
  5. 重复2,3,4直到low=high+1，返回array[high]

其中low总是指在第一个数组上，high总是指在第二个数组上，每次循环就是判断mid在哪边

问题

当array[mid]==array[0]时其实是存在问题的，例如输入为1011111和1111101时两个array[mid]都为1，不能够判断mid到底处于哪边

数组中出现次数超过一半的数字

题目描叙

数组中有一个数字出现的次数超过数组长度的一半，请找出这个数字。例如输入一个长度为9的数组{1,2,3,2,2,2,5,4,2}。
由于数字2在数组中出现了5次，超过数组长度的一半，因此输出2。如果不存在则输出0。

思路

  1. 在遍历数组时保存两个值：一个是数组中的一个数字，一个是次数
  2. 当我们遍历到一个数字时，如果count等于0，则用num记录改数子并且count+1
  3. 当count不为0且该数字等于num时，count+1
  4. 当count不为0且该数字不等于num时，count-1，此时若是count==0则把num的值替换为这个数字并给count+1
  5. 由于某个数字出现的次数比数组的长度的一半还多，所以最后一次修改num值的那个数字就是我们要找的那个数字
  6. 需要判断输入的数组是否满足题目的要求
